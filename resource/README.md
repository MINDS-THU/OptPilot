# Internal Resources

This directory is for OptPilot internal design notes and local development
resources.

Top-level Markdown files may be tracked when they describe internal architecture,
product decisions, or implementation plans. Public user-facing documentation
belongs under `docs/`.

Subdirectories are ignored by default and should be treated as local scratch
space, external codebases, or vendor material unless a specific subdirectory is
intentionally unignored. Do not rely on untracked resource subdirectories for
public documentation, tests, or packaged runtime behavior.
